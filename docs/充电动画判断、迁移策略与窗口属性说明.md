# 充电动画判断、迁移策略与窗口属性说明

本文基于 **MiRoot 1.8.2 当前实现**整理（流程对齐 MiRoot-3.4 老版，视觉与部分时序为 1.8.2 自有逻辑）。核心文件：

| 文件 | 职责 |
|------|------|
| `charging/ChargingService.java` | 插电/拔电、迁屏、官方背屏禁用、常亮唤醒循环 |
| `charging/RearScreenChargingActivity.java` | 主屏占位 / 背屏 UI、窗口属性、自动结束、背屏恢复 |
| `charging/ChargingOfficialSubscreen.kt` | 充电动画期间 force-stop 官方背屏中心 |
| `charging/ChargingAnimationPrefs.kt` | 开关、常亮、涨水速度 |
| `charging/ChargingIntents.java` | 包内显式广播 Action |
| `charging/RearChargingAnimationCoordinator.java` | 充电视与通知动画互斥、是否恢复背屏 |
| `charging/GyroWaterRippleView.java` | 陀螺仪水波液面与涟漪（1.8.2 视觉，非 3.4 闪电形状） |

---

## 1. 充电动画触发与结束判断

### 1.1 服务与广播入口

`ChargingService` 在 `onCreate()` 中：

- 绑定 `RootTaskService`（`ITaskService`）
- 注册电源广播：`ACTION_POWER_CONNECTED` / `ACTION_POWER_DISCONNECTED`
- 注册 `ACTION_RELOAD_CHARGING_SETTINGS`（功能页改设置后重读偏好）
- 注册 `ACTION_RESUME_CHARGING_ANIMATION`（通知等场景结束后恢复充电动画）
- 以前台服务形式常驻（通知与 `RearSwitchKeeperService` 共用样式）

### 1.2 插电触发（`onPowerConnected`）

依次判断：

1. **功能开关**：`ChargingAnimationPrefs.isEnabled()`，关闭则直接返回。
2. **冷却时间**：距上次成功触发须 ≥ `CHARGING_ANIMATION_COOLDOWN_MS`（**6000ms**）。
3. **电量**：`BatteryManager.BATTERY_PROPERTY_CAPACITY`。
4. **协调器**：`RearChargingAnimationCoordinator.startAnimation(CHARGING)`；若此前为通知动画，先发 `ACTION_INTERRUPT_NOTIFICATION_ANIMATION` 打断。
5. **迁屏流程**：`showChargingOnRearScreen(level, isLocked)`（见第 2 节）。
6. **常亮**：若 `ChargingAnimationPrefs.isAlwaysOn()` 为 true，在主线程启动 `startWakeupAndUpdateLoop()`。

**TaskService 未就绪**：`showChargingOnRearScreenWithRetry` 最多重试 **10 × 100ms**，超时则本轮放弃。

**流程 WakeLock**：迁屏线程内 `acquireWakeLock(8000ms)`，在 `finally` 中释放（`PARTIAL_WAKE_LOCK`）。

> 与旧版文档差异：当前 **无** `chargingFlowInProgress` 并发锁、**无** 插电防抖；拔电 **无** `POWER_DISCONNECT_DEBOUNCE_MS` 二次确认，收到 `ACTION_POWER_DISCONNECTED` 即结束动画。

### 1.3 拔电与结束

`onPowerDisconnected()`：

1. `stopWakeupLoop()`（停止常亮 800ms 唤醒循环）
2. `finishChargingAnimation()` → 发送 `ACTION_FINISH_CHARGING_ANIMATION`

`RearScreenChargingActivity` 收到后 **立即 `finish()`**。

### 1.4 Activity 内广播与生命周期

`finishReceiver` 注册的 Action：

| Action | 行为 |
|--------|------|
| `ACTION_FINISH_CHARGING_ANIMATION` | 拔电结束，`finish()` |
| `ACTION_INTERRUPT_CHARGING_ANIMATION` | 被新动画打断，`finish()` 且 **不恢复** 背屏（由协调器 `shouldRestoreOnDestroy=false`） |
| `ACTION_UPDATE_CHARGING_BATTERY` | `updateBatteryLevel`（常亮循环也会间接刷新 UI） |
| `ACTION_NOTIFY_CHARGING_TASK_MOVED_TO_REAR` | **迁屏完成后立即** `ensureChargingUiOnRearDisplay()` → `setupChargingUiOnRear` |

自动结束（对齐 3.4 时长，非旧文档 10s）：

| 模式 | 策略 |
|------|------|
| **普通模式** | 背屏 UI 与涨水启动后 **8s** 自动 `finish()`（`AUTO_FINISH_MS = 8000`） |
| **常亮模式** | `charging_always_on_enabled` 为 true 时不安排 8s 自动结束 |
| **onResume 补偿** | 背屏已显示但未安排过 auto-finish 且非常亮时，**5s** 后 `finish()`（`RESUME_COMPENSATE_FINISH_MS`） |

`ensureChargingUiOnRearDisplay()` 还会在 `onConfigurationChanged`、`onWindowFocusChanged`、`onResume` 中调用；若仍在主屏（`displayId == 0`）则 **不** inflate。

### 1.5 恢复充电动画

`ACTION_RESUME_CHARGING_ANIMATION`：延迟 300ms 后若仍插电且开关开启，再次 `showChargingOnRearScreen`；若常亮开启则重启唤醒循环。

## 2. 迁移（移动）策略

### 2.1 职责划分

- **迁屏由 `ChargingService` 统一完成**；`RearScreenChargingActivity` **不自迁移**。
- 目标：将 `RearScreenChargingActivity` 的 task 放到副屏（默认 **`displayId = 1`**）。
- Android 11+：先在 **主屏** 启动 Activity（透明占位、不 `setContentView`），再 shell 迁到背屏；迁屏成功后通过广播或生命周期在背屏 **inflate 真实 UI**。

### 2.2 插电迁屏完整顺序（当前实现）

`showChargingOnRearScreenWithRetry` 在 `taskService != null` 时执行：

```
1. resolveRearTaskIdForCharging()
      └─ 依据 SwitchToRearQsTileService.getLastMovedTask() 与背屏前台是否匹配
2. 若 rearTaskId > 0 → RearSwitchKeeperService.pauseMonitoring()
3. sendRearWakeupOnce(1)                    // 迁屏前点亮背屏
4. am start -n …/RearScreenChargingActivity
      --ei batteryLevel / rearTaskId / rearDisplayId
5. pollChargingTaskIdWithGrep()
      └─ am stack list | grep RearScreenChargingActivity
      └─ 最多 60×30ms；第 20、40 次重发 am start
6. service call activity_task 50 i32 <taskId> i32 1
      └─ sleep 40ms
7. confirmChargingTaskMovedToRear()
      └─ 最多 8×35ms，ITaskService.isTaskOnDisplay(taskId, 1)
      └─ 第 3、6 次失败时重发 move 命令
8. ChargingOfficialSubscreen.applyDisableBeforeChargingFlow()   // 迁屏后再禁用官方
9. sendRearWakeupOnce(1)
10. notifyChargingTaskMovedToRear(1)      // ACTION_NOTIFY_… 触发 inflate
```

**与 MiRoot-3.4 老版的关键差异**：

| 步骤 | 3.4 老版 | 1.8.2 当前 |
|------|----------|------------|
| 禁用官方背屏 | **迁屏前** `disableSubScreenLauncher()` | **迁屏确认后** `forceStopOfficialSubscreenForCharging()` |
| 迁屏确认 | 无 `isTaskOnDisplay` 轮询 | 有 `confirmChargingTaskMovedToRear` |
| 通知 inflate | 无 NOTIFY 广播 | **有** `ACTION_NOTIFY_CHARGING_TASK_MOVED_TO_REAR` |
| 背屏唤醒 | 迁屏前无单独唤醒 | 迁屏前、禁用后各 `KEYCODE_WAKEUP` 一次 |

> **为何迁屏后再 disable**：主屏占位阶段 Activity 尚未 `setContentView`；若先 force-stop 官方背屏，背屏在 UI inflate 前会长时间全黑。NOTIFY 用于部分 ROM 上 `getDisplay()` / 配置变更滞后，避免「快结束才闪一下界面」。

### 2.3 背屏 taskId 解析（用于结束后恢复投屏 app）

`resolveRearTaskIdForCharging()`：

- 读取 `SwitchToRearQsTileService.getLastMovedTask()`（格式含 `package:taskId`）
- 用 `taskService.getForegroundAppOnDisplay(1)` 判断：
  - 背屏前台为 `RearScreenChargingActivity`（上一轮动画未销毁），或
  - 背屏前台与 `lastTask` 一致（投送 app 仍在背屏）
- 满足则解析出 `rearTaskId` 写入 Intent，供 `onDestroy` 恢复投送 task

**未解析到**（`rearTaskId == -1`）：结束后走恢复官方副屏桌面。

### 2.4 官方背屏服务禁用与恢复

**禁用**（`ChargingOfficialSubscreen.applyDisableBeforeChargingFlow`）：

- 调用 `ITaskService.forceStopOfficialSubscreenForCharging()`（等同 `am force-stop com.xiaomi.subscreencenter`）
- 与功能页「禁用官方背屏」总开关无关；`disabledByCharging` 防止同一次动画重复 force-stop

**结束后恢复**（`RearScreenChargingActivity.onDestroy`，且 `RearChargingAnimationCoordinator.endAnimation` 返回 `shouldRestore=true`）：

| 情况 | 行为 |
|------|------|
| `rearTaskId > 0` | `restoreRearProjectedTask`：再次 force-stop 官方 → `service call activity_task 50` 迁回投送 task（可能执行两次）→ `restartKeeperService()`；**不**调用 `enableSubScreenLauncher()` |
| `rearTaskId <= 0` | `restoreRearOfficialLauncher()`：`am start --display <id> com.xiaomi.subscreencenter/…SubScreenLauncherActivity` |

`ChargingOfficialSubscreen.restoreAfterChargingFlow()` / `reviveOfficialPackageAfterChargingWithoutLauncher()` 仍保留在代码中，供其他链路或后续接入；**当前充电动画结束路径以 Activity 内 shell 恢复为主**。

### 2.5 常亮模式下的背屏保活

仅当 `ChargingAnimationPrefs.KEY_ALWAYS_ON` 为 true：

- `startWakeupAndUpdateLoop()`：每 **800ms** 向 display 1 发送 `KEYCODE_WAKEUP`，并 `RearScreenChargingActivity.updateBatteryLevelStatic`
- 每 50 次循环检查背屏前台是否仍为 `RearScreenChargingActivity`，否则停止循环
- 与 3.4 一致：**普通模式不在 Service 层做周期唤醒**（仅靠窗口 `FLAG_KEEP_SCREEN_ON` 与迁屏前后各一次唤醒）

### 2.6 当前未使用的迁屏相关能力

以下在 **当前 `ChargingService` 中未调用**（旧文档或历史分支曾描述）：

- `RearAssistService.pauseMonitoringForCharging` / `RearProjectionProximityGate`
- 锁屏失败后的 `LOCKSCREEN_DEFERRED_RETRY_MS` 二次迁屏
- `am task move-to-front`、应用侧解析 `am stack list`（无 grep 管道）
- `waitChargingForegroundOnRear` 前台校验后才 NOTIFY
- `ChargingPlaceholderActivity`（Manifest 已注册，启动链路仍用 `RearScreenChargingActivity` 主屏占位）

## 3. 窗口属性与显示策略

### 3.1 Manifest（`RearScreenChargingActivity`）

- `launchMode="singleInstance"`，`taskAffinity="com.wmqc.miroot.charging"`
- `theme="@style/Theme.MiRoot.Transparent.NoPreview"`（主屏占位透明）
- `showWhenLocked`、`turnScreenOn`、`hardwareAccelerated`
- `configChanges` 含 `density` 等，迁屏后可能 **不销毁实例** 而仅配置变更
- `meta-data miui.rear.policy = miui`

### 3.2 主屏占位 vs 背屏渲染

**Android 11+（`onCreate`）**：

- `getCurrentDisplayIdSafe() == 0`：仅 `registerFinishReceiverOnly()` + `currentInstance = this`，**不** `setContentView`、**不** 加背屏窗口标志
- 非默认显示屏：走 `setupChargingUiOnRear()` 完整 UI

**背屏 inflate（`setupChargingUiOnRear`）**：

1. `RearDisplayInputHelper.ensureApplicationWindowReceivesInput`
2. `applyRearChargingWindowFlags()`
3. `applyRearDensityBeforeInflateIfPossible()`（按 `getCurrentRearDpi()` 对齐密度）
4. `getWindow().setFormat(OPAQUE)` + `setBackgroundDrawableResource(R.color.charging_window_bg)`
5. `setContentView(R.layout.activity_rear_screen_charging)`
6. `bindChargingViews` → `startFillAndBatteryPresentation` → `hideSystemUi()`

保证：**主屏不出现充电动画布局**；背屏加载前有不透明窗口底色，减轻纯黑。

### 3.3 窗口标志（`applyRearChargingWindowFlags`）

- `FLAG_KEEP_SCREEN_ON`
- `FLAG_SHOW_WHEN_LOCKED` / `FLAG_TURN_SCREEN_ON`
- API 27+：`setShowWhenLocked(true)`、`setTurnScreenOn(true)`
- `FLAG_HARDWARE_ACCELERATED`
- API 28+：`LAYOUT_IN_DISPLAY_CUTOUT_MODE_ALWAYS`

`onResume` 会再次叠加 KEEP_SCREEN_ON / SHOW_WHEN_LOCKED / TURN_SCREEN_ON。

### 3.4 沉浸式与安全区

- `hideSystemUi()`：API 30+ 使用 `WindowInsetsController` 隐藏状态栏与导航栏；decor 为 0×0 时最多延迟重试 6 次（16ms），避免迁屏首帧 Insets 刷屏
- 电量 `TextView`：`WindowInsets`（cutout + systemBars）转 padding，背景 `GyroWaterRippleView` 仍全屏；17 Pro 等有 `REAR_CAMERA_FALLBACK_LEFT_PX` 保底左避让

### 3.5 布局与字号

`activity_rear_screen_charging.xml`：

- 全屏 `GyroWaterRippleView` + 居中 `battery_text`（**60sp**，对齐 3.4 布局量级）
- 容器背景 `@color/charging_window_bg`

## 4. 涨水动画与视觉（1.8.2 自有，非 3.4 液面曲线）

### 4.1 涨水时长

由 `ChargingAnimationPrefs` 控制，**线性** `ValueAnimator`（**非** 3.4 固定 2s 减速曲线）：

- `KEY_FILL_RISE_SPEED_PERCENT`：40～80（默认 50）
- 满刻度时长：`fillDurationMsForFullFill(speedPercent) = FILL_MS_FOR_FULL_SCALE * 100 / speedPercent`（`FILL_MS_FOR_FULL_SCALE = 3200`）
- 实际时长：`durationMs = fullFillMs * targetLevel`（按当前电量比例）

首段涨水期间 `initialFillAnimating=true`，外部电量更新只改目标值，不直接跳液位。

电量文字：600ms 延迟后 800ms 淡入放大（`DecelerateInterpolator`）。

### 4.2 `GyroWaterRippleView` 运动与涟漪

- 传感器：陀螺仪 + 加速度计
- `Choreographer` 驱动 `tick()`：`wavePhase`、`liquidShiftX/Y` 随陀螺与重力偏移，带衰减与限幅
- 自动涟漪：常规间隔 **1.8s**，剧烈晃动 **0.72s**（滞回阈值 `RIPPLE_HIGH_MOTION_ENTER/EXIT`）
- 液面颜色随电量阈值（约 20%）绿/橙切换；最多 **96** 个 ripple 实例
- `onAttachedToWindow` 会先画一帧，减轻 Surface 就绪慢时的黑屏

该部分 **仅影响视觉**，不参与迁屏或结束判断。

## 5. 广播 Action 一览

| Action | 发送方 | 作用 |
|--------|--------|------|
| `ACTION_FINISH_CHARGING_ANIMATION` | `ChargingService`（拔电） | 结束充电动画 |
| `ACTION_INTERRUPT_CHARGING_ANIMATION` | `RearChargingAnimationCoordinator` | 打断充电动画且不恢复背屏 |
| `ACTION_UPDATE_CHARGING_BATTERY` | 外部/常亮循环 | 更新电量显示 |
| `ACTION_NOTIFY_CHARGING_TASK_MOVED_TO_REAR` | `ChargingService`（迁屏后） | 立即在背屏 inflate UI |
| `ACTION_RELOAD_CHARGING_SETTINGS` | 功能页 | 重读常亮等偏好 |
| `ACTION_RESUME_CHARGING_ANIMATION` | 通知等场景 | 恢复充电动画 |

均通过 `intent.setPackage(getPackageName())` 包内投递。

## 6. 与 MiRoot-3.4 对齐关系速查

| 项目 | 对齐 3.4 | 1.8.2 保留差异 |
|------|----------|----------------|
| 冷却 6s、Keeper 暂停、grep 轮询 taskId、move 50、8s 自动结束 | ✓ | — |
| 常亮 800ms 唤醒 + 电量刷新 | ✓ | 常亮开关在偏好中（功能页可隐藏） |
| 结束后恢复投屏 task / 官方 Launcher + 重启 Keeper | ✓ | 恢复投屏时仍 force-stop 官方 |
| 禁用官方时机 | 3.4：**迁屏前** | 1.8.2：**迁屏后** + NOTIFY |
| 充电视觉 | 3.4：闪电/老液面 | **GyroWaterRippleView** + 可调涨水速度 |
| 电量字号 | 3.4 量级 | **60sp** |
