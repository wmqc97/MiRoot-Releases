# MiRoot 离线激活说明

## 1. 设备码来源说明

MiRoot 进入未激活状态时会弹出“离线激活”弹窗，界面中显示的 `设备码` 是由本机多项系统标识组合后再计算得到的结果。

当前设备码来源为下列 **稳定 Build 字段**（不含 `ANDROID_ID`、`FINGERPRINT`），顺序固定、以 `|` 拼接后再做摘要：

- `Build.BRAND`
- `Build.MANUFACTURER`
- `Build.MODEL`
- `Build.DEVICE`
- `Build.HARDWARE`
- `Build.BOARD`
- `Build.PRODUCT`

这样同一物理机在 **重装应用**、切换 **debug / release 签名** 时设备码通常保持不变。

### 说明

- 设备码不是单一硬件序列号，而是上述字段的摘要。
- 若厂商 OTA 修改了机型相关 Build 字段，或刷机 / 修改机型伪装，设备码仍可能变化。
- 建议用户在设备环境稳定后再申请激活码。

## 2. 激活码计算公式（与当前代码一致）

MiRoot 本身不提供本机生成激活码界面，但内部校验逻辑与 `generate_activation_code.py` 一致。  
同一设备码始终对应同一激活码（永久固定码）。

### 2.1 输入归一化

设备码 `deviceCode` 先做归一化：

- `trim()`
- 转大写
- 去除空白字符（空格、换行、Tab 等）

记归一化后的结果为 `DC`。

### 2.2 KDF 参数

使用 `PBKDF2WithHmacSHA256`，参数如下：

- `iterations = 80000`
- `dkLen = 32 bytes`
- `password = "MiRoot-Permanent|" + DC + "|" + INTERNAL_SALT`
- `salt = "MiRoot-Offline-Activation|" + DC + "|v2"`
- `INTERNAL_SALT = "c8c4b7c9" + "fa3d91e2" + "5a7f29bd"`

即：

`derived = PBKDF2_HMAC_SHA256(password, salt, 80000, 32)`

### 2.3 输出格式

1. `derived` 转为 64 位十六进制大写字符串（`[0-9A-F]`）。
2. 每 8 位分组，以 `-` 连接。

输出示例形态：

`XXXXXXXX-XXXXXXXX-XXXXXXXX-XXXXXXXX-XXXXXXXX-XXXXXXXX-XXXXXXXX-XXXXXXXX`

共 8 组，每组 8 位，总有效字符 64（不含分隔符）。

## 3. 激活校验方式

校验规则为：

1. 将用户输入激活码归一化为大写，并去除非字母数字字符（空白、短横线等都会被忽略）。
2. 使用当前设备码按上述公式重新计算目标激活码。
3. 两者完全一致则校验通过。
4. 校验通过后，应用写入本地激活标记密文（AES/GCM），用于后续已激活状态判断。

## 4. 使用说明

- 用户需要把当前界面显示的设备码提供给激活码生成方。
- 激活码生成方根据设备码生成对应激活码后，用户再回填到应用中。
- 用户输入时可以包含空格或短横线，系统会自动忽略。

## 5. 激活状态说明

MiRoot 在“是否已激活”的判断中，不仅看本地布尔值，还会校验本地保存的激活证明信息。
仅修改本地状态标记不足以绕过激活。
