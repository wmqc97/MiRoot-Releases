# 酷我车载 LRCX 歌词跨应用广播 — 参考文档

本文档说明 **7.3.9.23 补丁树** 中新增的 **LRCX 逐字歌词对外广播** 协议、实现位置与接收方接入方式。与 [酷我移植参考文档.md](./酷我移植参考文档.md) 中的悬浮歌词、插件设置相互独立，可单独使用。

---

## 1. 功能概览

| 项目 | 说明 |
|------|------|
| **发送方** | 酷我车载 `cn.kuwo.kwmusiccar` |
| **独立开关** | 插件设置 → **「歌词对外广播」**（prefs `lyric_export_enabled`，默认关）；**无需**开启悬浮歌词或自动跳转 |
| **启动条件** | **仅**在 `lyric_export_enabled=true` 时：`syncService` 拉起 `FloatLyricService`（若悬浮/自动跳转均未开，则只跑广播、不建悬浮窗）；`scheduleNextExport` / `tick` 均会二次校验该开关 |
| **轮询** | 开关开启后，服务内每 **200ms** 调用一次 `LyricExportBroadcast.tick`（内部检测，不一定每次都发广播） |
| **全量歌词** | 每首歌 **至多一条** `LYRIC_FULL`：等歌词对象就绪后一次性带上 `lines[]` / `line_times_ms[]`（及 LRCX 的 `words_json`），**不会**先发空全量再补完整全量 |
| **进度** | 仅 `LYRIC_PROGRESS` 更新当前行/进度/播放位置，**不含**整首歌词行数组 |
| **LRCX** | 全量带 `words_json`；进度带 `is_lrcx` 与 `line_progress`（0–100） |
| **普通 LRC** | 全量仅 `lines` + `line_times_ms`；进度 `is_lrcx=false` |

### 1.1 开关行为

| 操作 | 行为 |
|------|------|
| **打开** | `resetForExport()` 清空缓存 → `syncService` 启服务 → **立即 `tick` 一次**（当前在播曲目会马上尝试发全量） |
| **关闭** | `resetForExport()`；若无悬浮歌词/自动跳转则 `stopService`，否则 `onStartCommand` 移除导出 `Handler` 回调 |

## 2. 广播 Action

| Action | 时机 |
|--------|------|
| `cn.kuwo.kwmusiccar.action.LYRIC_FULL` | 见 §3.3（每曲一次为主；歌词晚到补一条） |
| `cn.kuwo.kwmusiccar.action.LYRIC_PROGRESS` | 见 §4.1 节流规则 |

均为 **隐式广播**（未 `setPackage`），跨应用接收方须使用 **`Context.registerReceiver`** 动态注册（Android 8–16 对 manifest 静态接收自定义 Action 有限制）。

## 3. 全量广播 `LYRIC_FULL`

### 3.1 Extra 字段

| Extra | 类型 | 说明 |
|-------|------|------|
| `music_id` | `long` | `Music.f` 歌曲 ID |
| `title` | `String` | `Music.g`，空则 `""` |
| `artist` | `String` | `Music.h`，空则 `""` |
| `duration_ms` | `long` | `PlayProxy.getDuration()` |
| `lyric_type` | `String` | `LRCX` / `LRC` / `TXT` |
| `lines` | `String[]` | 全部歌词行文本（`a.c()`） |
| `line_times_ms` | `int[]` | 各行起始时间 ms（`a.b()`，与行下标对齐） |
| `words_json` | `String` | **仅 `lyric_type=LRCX`** 且实现类为 `cn.kuwo.mod.lyric.m` 时存在 |

### 3.2 `words_json` 格式

二维 JSON 数组：外层下标 = **行号**，内层 = 该行逐字/词单元。

```json
[
  [
    {"cs":0,"ce":2,"wi":0,"s":12340,"e":12500},
    {"cs":2,"ce":4,"wi":1,"s":12500,"e":12800}
  ],
  []
]
```

| 字段 | 来源 `LyricsDefine$c` | 含义 |
|------|----------------------|------|
| `cs` | `a` | 行内字符起始下标 |
| `ce` | `b` | 行内字符结束下标 |
| `wi` | `c` | 词序号 |
| `s` | `d` | 词起始时间（ms，相对曲内时间轴） |
| `e` | `e` | 词结束时间（ms） |

某字段为 `null` 时该键**省略**（不写入 JSON 对象）。

### 3.3 何时发送（每曲一次）

```text
切歌 → 更新 sLastMusicId，重置 sLastLyricId=0
       ↓
歌词对象 a 就绪 且 identityHashCode ≠ sLastLyricId
       ↓
发送一条 LYRIC_FULL（含完整 lines / times / words_json）
       ↓
记录 sLastLyricId，同一歌词实例不再重复全量
```

- **歌词未加载**：不发空全量，等 `l2()` 返回实例后再发。  
- **歌词晚到**（切歌后异步加载）：补发 **一条** 完整全量。  
- **打开广播开关**：`resetForExport()` 后等同重新切歌，当前曲会再发一条全量。

## 4. 进度广播 `LYRIC_PROGRESS`

| Extra | 类型 | 说明 |
|-------|------|------|
| `music_id` | `long` | 当前曲 ID |
| `title` / `artist` | `String` | 同上 |
| `position_ms` | `long` | `getCurrentPos() + r0.B()` |
| `line_index` | `int` | 当前行下标（`LyricsDefine.b.a`），无匹配为 `-1` |
| `line_text` | `String` | 当前行文案 |
| `line_progress` | `int` | 行内进度 **0–100**（仅 LRCX 有效） |
| `is_lrcx` | `boolean` | 是否 LRCX 逐字 |
| `lyric_type` | `String` | `LRCX` / `LRC` / `TXT` |
| `playing` | `boolean` | `PlayProxy.Status == PLAYING`（7.3.9 为 `Status.c`） |

**不含** `lines`、`line_times_ms`、`words_json`；接收方应缓存 `LYRIC_FULL`，进度只更新 UI。

### 4.1 发送节流

满足 **任一** 条件才发 `LYRIC_PROGRESS`（否则本 tick 跳过）：

| 条件 | 说明 |
|------|------|
| 播放状态变化 | `Status` 与上次不同（含暂停/播放切换） |
| 进度位移 | 距上次**已发送**进度，\|Δposition_ms\| ≥ **200ms** |
| 行号变化 | `line_index` 与上次不同 |
| 行内进度变化 | `line_progress` 与上次不同（LRCX 逐字） |

- 内部轮询 **200ms** 一次；**不是**播放中每 tick 必发。  
- 仅在**实际发送**后更新 `sLastExportPos` / `sLastLineIndex` / `sLastLineProgress`；`sLastStatus` 每 tick 同步。  
- 异常在 `tick` 内静默忽略，不 `printStackTrace`，避免 logcat 刷屏。

## 5. 实现结构

| 文件 | 职责 |
|------|------|
| `smali_classes2/.../LyricExportBroadcast.smali` | 组包、`sendBroadcast`、`resetForExport`、`tick` 节流、LRCX `words_json` |
| `smali_classes2/.../FloatLyricService$5.smali` | 200ms `tick` + `scheduleNextExport` |
| `smali_classes2/.../FloatLyricService.smali` | `mExportRunnable`、`scheduleNextExport`、`onStartCommand` 按开关启停导出循环 |
| `smali_classes2/.../FloatLyricHelper.smali` | `isLyricExportEnabled`、`syncService`（三开关 OR 启服务） |
| `smali_classes2/.../PluginFloatLyricBinder$LyricExportClickListener.smali` | 开关切换 → `resetForExport` + `syncService` + 开启时 `tick` |
| `res/layout/activity_plugin_settings.xml` | 「歌词对外广播」行 |
| `tools/gen_lyric_export_broadcast.py` | smali 生成脚本（与现网手改 smali 可能略有偏差，改协议后优先改脚本再生成） |

### 5.1 数据读取（与悬浮歌词一致）

```text
Lc6/b;->n()Lg8/e;           # 播放代理
getCurrentPos() + Lq5/r0;->B()
Lc6/a;->g()Lcn/kuwo/mod/lyric/b;
l2()Lcn/kuwo/mod/lyric/a;
a.d(J, LyricsDefine$b)      # b.a=行号, b.b=行内0-100（LRCX）
a.c()                       # 行文本列表（勿用 a.b() 当文案）
getType() == LyricsType.c   # LRCX
```

LRCX 逐字列表：`cn.kuwo.mod.lyric.m` 私有字段 **`h`** → `List<List<LyricsDefine$c>>`。

### 5.2 静态缓存

| 字段 | 用途 |
|------|------|
| `sLastMusicId` | 切歌检测；变化时重置 `sLastLyricId` |
| `sLastLyricId` | `identityHashCode(lyric)`；变化且歌词非空时发 **一条** 全量 |
| `sLastExportPos` / `sLastLineIndex` / `sLastLineProgress` | 进度节流（**仅发送成功后** 写入） |
| `sLastStatus` | 播放状态变化检测 |
| `sStage*` | `sendStagedProgress` 暂存，避免 smali 寄存器超限 |

`resetForExport()`：开关切换时将所有缓存置初值（`sLastMusicId=-1` 等）。

无 `Context` 长期持有；`words_json` **仅在** `LYRIC_FULL` 时构建。

### 5.3 与悬浮歌词关系

| 开关组合 | 服务 | 悬浮窗 | 广播 |
|----------|------|--------|------|
| 仅 `lyric_export_enabled` | 启动 | 不创建 | 200ms tick |
| 仅 `enabled`（悬浮） | 启动 | 创建 | 不 tick（export 关） |
| 两者都开 | 启动 | 创建 | 200ms tick |
| 都关 | `stopService` | — | — |

## 6. 接收方示例（Kotlin）

```kotlin
class LyricReceiver : BroadcastReceiver() {
  private var cachedMusicId: Long = -1L
  private var lines: Array<String> = emptyArray()
  private var lineTimesMs: IntArray = intArrayOf()
  private var wordsJson: String? = null

  override fun onReceive(context: Context, intent: Intent) {
    when (intent.action) {
      "cn.kuwo.kwmusiccar.action.LYRIC_FULL" -> {
        cachedMusicId = intent.getLongExtra("music_id", -1L)
        lines = intent.getStringArrayExtra("lines") ?: emptyArray()
        lineTimesMs = intent.getIntArrayExtra("line_times_ms") ?: intArrayOf()
        wordsJson = intent.getStringExtra("words_json") // LRCX only
        // 缓存整首歌词；勿在 onReceive 里 Log 大数组
      }
      "cn.kuwo.kwmusiccar.action.LYRIC_PROGRESS" -> {
        val pos = intent.getLongExtra("position_ms", 0L)
        val line = intent.getIntExtra("line_index", -1)
        val text = intent.getStringExtra("line_text") ?: ""
        val progress = intent.getIntExtra("line_progress", 0)
        val lrcx = intent.getBooleanExtra("is_lrcx", false)
        val playing = intent.getBooleanExtra("playing", false)
        // 用 cached lines + 进度更新 UI；勿每条 progress 打 Log
      }
    }
  }
}

// Application.onCreate:
val filter = IntentFilter().apply {
  addAction("cn.kuwo.kwmusiccar.action.LYRIC_FULL")
  addAction("cn.kuwo.kwmusiccar.action.LYRIC_PROGRESS")
}
if (Build.VERSION.SDK_INT >= 33) {
  registerReceiver(LyricReceiver(), filter, Context.RECEIVER_EXPORTED)
} else {
  registerReceiver(LyricReceiver(), filter)
}
```

**建议**：全量在 `LYRIC_FULL` 缓存；`LYRIC_PROGRESS` 只读当前行字段。调试时用 `music_id` / `line_index` 过滤，避免对每条广播 `Log.d` 导致 logcat 刷屏。

## 7. 兼容与限制

| 项 | 说明 |
|----|------|
| Android 9–16 | 动态 `registerReceiver` 可收；发送方 targetSdk 34 无额外权限 |
| 仅 LRCX | `words_json` / 有意义之 `line_progress` |
| 普通 LRC | 行级 `lines` + `line_times_ms`，`line_progress` 可忽略 |
| TXT | 无时间轴，`line_times_ms` 可能为空 |
| 开关未开 | 不 `tick`、不 `scheduleNextExport`、不发任何广播 |
| Bundle 大小 | 超长 LRCX 全量可能较大；务必缓存全量、进度只跟 `LYRIC_PROGRESS` |
| 进度频率 | 约 ≤5 次/秒（200ms 轮询 + 节流），LRCX 行内进度变化时可能更密 |

## 8. 验收清单

1. **仅开「歌词对外广播」**（悬浮/自动跳转关）→ 无悬浮窗；播放 LRCX → 收到 `LYRIC_FULL` + 节流后的 `LYRIC_PROGRESS`。
2. **打开开关瞬间**（正在播放）→ 当前曲 **一条** `LYRIC_FULL`（含 `lines`），非空数组（歌词已加载时）。
3. **切歌** → 新歌 **一条** `LYRIC_FULL`；不出现「先空 `lines=[]` 再完整」两条全量。
4. **播放中** → `LYRIC_PROGRESS` 约 2–5 次/秒（非 10 次/秒）；`line_progress`（LRCX）随播放变化。
5. **暂停 / seek** → 收到 `playing=false` 或 `position_ms` 跳变相关进度。
6. **关闭开关** → 停止新广播；仅悬浮开时服务仍可存活但无 `LYRIC_*`。
7. **普通 LRC** → `LYRIC_FULL` 无 `words_json`，`is_lrcx=false`。
