# MiRoot 外部广播：音乐投屏 / 应用投屏 / 背屏桌面 / 车控投屏 / 激活状态查询

第三方应用、Tasker、MacroDroid、ADB 等可通过广播启动或停止背屏投屏、在背屏打开**背屏桌面**，并可**查询**音乐投屏当前状态、订阅**状态变更回调**，以及**查询**本机 MiRoot 是否已完成离线激活。下表为**投屏、背屏桌面与激活查询**的 Action、Extra 与示例；车控**远程指令、车辆状态查询**见 [车控广播调用 API 文档](./车控广播调用%20API%20文档.md)。

**通用约定**

| 项 | 值 |
|----|-----|
| 包名 | `com.wmqc.miroot` |
| 建议 | `Intent.setPackage("com.wmqc.miroot")`，减少 Android 12+ 隐式广播歧义 |

**前置（简要）**  
音乐投屏：需 Root 或 Shizuku、通知监听等，与应用内「音乐投屏」一致。  
应用投屏：需 Root 或 Shizuku，与快捷设置「切应用到背屏」磁贴一致（迁屏 + Keeper）。  
背屏桌面：无需 Root；需 Android 8.0+，且副屏（通常为 `displayId=1`）存在并已点亮，与应用内「在背屏打开桌面」一致。  
车控投屏：需 Root 或 Shizuku，与应用内「启动车控投屏」一致。

## 一、音乐投屏

| 项 | 值 |
|----|-----|
| 接收器（可显式 `-n`） | `com.wmqc.miroot.lyrics.LyricsMusicProjectionReceiver` |
| 清单 | `android:exported="true"` |

### Action（必填）

```
com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION
```

### Extra（可选）

| Key | 类型 | 说明 |
|-----|------|------|
| `com.wmqc.miroot.lyrics.EXTRA_MUSIC_PROJECTION_OP` | String | **不传或值为空**：**一键切换**——若背屏歌词投屏界面已在运行则停止，否则启动（与 App 内状态判定一致）。**显式** `start` / `stop`：固定开始或停止，兼容旧集成。 |

同一 Action，无 Extra 时反复发送即可在启停之间切换。

### 常量（与 `LyricsIntents` 一致）

| 含义 | 字符串 |
|------|--------|
| Action | `com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION` |
| Extra 键 | `com.wmqc.miroot.lyrics.EXTRA_MUSIC_PROJECTION_OP` |
| 开始 | `start` |
| 停止 | `stop` |

### 状态查询（不启停投屏）

向同一接收器发送查询 Action，MiRoot 以**回执广播**返回当前背屏歌词投屏是否活跃（与车控「车辆状态查询」用法类似：可选 `requestId`、`replyAction`）。

| 项 | 值 |
|----|-----|
| 请求 Action（必填） | `com.wmqc.miroot.lyrics.ACTION_QUERY_MUSIC_PROJECTION_STATUS` |
| 默认回执 Action | `com.wmqc.miroot.lyrics.ACTION_MUSIC_PROJECTION_STATUS_RESULT` |

**请求 Extra（可选）**

| Key | 类型 | 说明 |
|-----|------|------|
| `requestId` | String | 回执中原样带回 |
| `replyAction` | String | 自定义回执 Action；不传则用默认 |

**回执 Extra（MiRoot 发出）**

| Key | 类型 | 说明 |
|-----|------|------|
| `requestId` | String | 与请求一致 |
| `success` | String | `"true"` / `"false"`（查询本身是否成功；当前实现成功时为 `"true"`） |
| `running` | String | 背屏歌词界面是否活跃：`"true"` / `"false"`（与车控 `success` 同为字符串，便于 MAML） |

### 状态变更回调（被动）

当音乐投屏开始或结束（背屏歌词界面出现或关闭）时，MiRoot 会**主动**发送一条广播（**不**调用 `setPackage`，第三方进程可注册接收）。

| 项 | 值 |
|----|-----|
| Action | `com.wmqc.miroot.lyrics.ACTION_MUSIC_PROJECTION_STATE_CHANGED` |

**Extra**

| Key | 类型 | 说明 |
|-----|------|------|
| `running` | String | `"true"`：背屏歌词投屏活跃；`"false"`：已结束 |

应用内页面仍使用包内广播 `com.wmqc.miroot.internal.MUSIC_PROJECTION_STATE`（boolean）；外部集成请使用上表 Action 与字符串 `running`。

### ADB 示例

```bash
# 一键切换（无 Extra：已在投屏则停，否则开）
adb shell am broadcast -a com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION com.wmqc.miroot
```

```bash
# 显式接收器（同样为一键切换）
adb shell am broadcast -n com.wmqc.miroot/com.wmqc.miroot.lyrics.LyricsMusicProjectionReceiver \
  -a com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION
```

```bash
# 显式仅停止（不依赖当前状态）
adb shell am broadcast -a com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION \
  --es com.wmqc.miroot.lyrics.EXTRA_MUSIC_PROJECTION_OP stop \
  com.wmqc.miroot
```

```bash
# 显式仅开始（不依赖当前状态）
adb shell am broadcast -a com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION \
  --es com.wmqc.miroot.lyrics.EXTRA_MUSIC_PROJECTION_OP start \
  com.wmqc.miroot
```

```bash
# 查询投屏状态（回执默认发往 com.wmqc.miroot.lyrics.ACTION_MUSIC_PROJECTION_STATUS_RESULT）
adb shell am broadcast -a com.wmqc.miroot.lyrics.ACTION_QUERY_MUSIC_PROJECTION_STATUS \
  --es requestId q1 \
  com.wmqc.miroot
```

```bash
# 显式接收器（查询）
adb shell am broadcast -n com.wmqc.miroot/com.wmqc.miroot.lyrics.LyricsMusicProjectionReceiver \
  -a com.wmqc.miroot.lyrics.ACTION_QUERY_MUSIC_PROJECTION_STATUS
```

### 应用内调用示例

**Kotlin**

```kotlin
import android.content.Intent

/** 一键切换：与仅 Action、无 Extra 的 ADB/主题广播一致 */
fun toggleMusicProjection(ctx: android.content.Context) {
    ctx.sendBroadcast(
        Intent("com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION").apply {
            setPackage("com.wmqc.miroot")
        },
    )
}

fun startMusicProjection(ctx: android.content.Context) {
    ctx.sendBroadcast(
        Intent("com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION").apply {
            setPackage("com.wmqc.miroot")
            putExtra("com.wmqc.miroot.lyrics.EXTRA_MUSIC_PROJECTION_OP", "start")
        },
    )
}

fun stopMusicProjection(ctx: android.content.Context) {
    ctx.sendBroadcast(
        Intent("com.wmqc.miroot.lyrics.ACTION_OPEN_MUSIC_PROJECTION").apply {
            setPackage("com.wmqc.miroot")
            putExtra("com.wmqc.miroot.lyrics.EXTRA_MUSIC_PROJECTION_OP", "stop")
        },
    )
}

fun queryMusicProjectionStatus(ctx: android.content.Context, requestId: String? = null) {
    ctx.sendBroadcast(
        Intent("com.wmqc.miroot.lyrics.ACTION_QUERY_MUSIC_PROJECTION_STATUS").apply {
            setPackage("com.wmqc.miroot")
            putExtra("requestId", requestId ?: "")
        },
    )
}
```

主题、Tasker、MacroDroid 等可注册接收 `com.wmqc.miroot.lyrics.ACTION_MUSIC_PROJECTION_STATE_CHANGED`，读取字符串 extra `running`（`"true"` / `"false"`）。

**Java**

```java

void stopMusicProjection(Context context) {
```

## 二、应用投屏（主屏前台应用迁到背屏）

将**当前主屏前台应用**迁到背屏，行为与快捷设置磁贴「切应用到背屏」一致（含 `RearSwitchKeeperService` 与迁回通知）。

| 项 | 值 |
|----|-----|
| 接收器（可显式 `-n`） | `com.wmqc.miroot.rear.AppProjectionBroadcastReceiver` |
| 清单 | `android:exported="true"` |

### Action（必填）

```
com.wmqc.miroot.rear.ACTION_APP_PROJECTION
```

### Extra（可选）

| Key | 类型 | 说明 |
|-----|------|------|
| `com.wmqc.miroot.rear.EXTRA_APP_PROJECTION_OP` | String | `start`：迁屏（默认）；`stop`：请求迁回主屏（等价于点按迁屏 Keeper 通知，需 Keeper 正在运行）。 |

### 常量（与 `AppProjectionIntents` 一致）

| 含义 | 字符串 |
|------|--------|
| Action | `com.wmqc.miroot.rear.ACTION_APP_PROJECTION` |
| Extra 键 | `com.wmqc.miroot.rear.EXTRA_APP_PROJECTION_OP` |
| 开始 | `start` |
| 停止 | `stop` |

### ADB 示例

```bash
# 将主屏前台应用迁到背屏
adb shell am broadcast -a com.wmqc.miroot.rear.ACTION_APP_PROJECTION com.wmqc.miroot
```

```bash
adb shell am broadcast -n com.wmqc.miroot/com.wmqc.miroot.rear.AppProjectionBroadcastReceiver \
  -a com.wmqc.miroot.rear.ACTION_APP_PROJECTION
```

```bash
# 停止：迁回主屏（与 Keeper 通知点击相同，需迁屏 Keeper 已启动）
adb shell am broadcast -a com.wmqc.miroot.rear.ACTION_APP_PROJECTION \
  --es com.wmqc.miroot.rear.EXTRA_APP_PROJECTION_OP stop \
  com.wmqc.miroot
```

### 应用内调用示例（Kotlin）

```kotlin

            putExtra("com.wmqc.miroot.rear.EXTRA_APP_PROJECTION_OP", "start")
        },
    )
}

fun stopAppProjectionReturnMain(ctx: android.content.Context) {
    ctx.sendBroadcast(

            setPackage("com.wmqc.miroot")
            putExtra("com.wmqc.miroot.rear.EXTRA_APP_PROJECTION_OP", "stop")
        },
    )
}
```

## 四、车控投屏（吉利背屏车控界面）

| 项 | 值 |
|----|-----|
| 接收器（可显式 `-n`） | `com.wmqc.miroot.car.CarControlBroadcastReceiver` |
| 清单 | `android:exported="true"` |

车控**指令与状态查询**仍可使用文档中的 tgwgroup / MiRoot 双命名空间；**仅投屏**统一为下方 MiRoot Action。

### 4.1 打开投屏（同一 Action；无 Extra 时一键切换）

**Action**

```
com.wmqc.miroot.car.ACTION_OPEN_CAR_CONTROL_PROJECTION
```

**Extra（可选）**

| Key | 类型 | 说明 |
|-----|------|------|
| `com.wmqc.miroot.car.EXTRA_CAR_PROJECTION_OP` | String | **不传或值为空**：**一键切换**——若背屏车控界面已在运行则停止，否则启动。**显式** `start` / `stop`：固定开始或停止。 |

### 4.2 专用「仅停止投屏」Action（无需 Extra）

```
com.wmqc.miroot.car.ACTION_STOP_CAR_CONTROL_PROJECTION
```

### 常量一览（与 `CarControlIntents` 一致）

| 名称 | 值 |
|------|-----|
| `ACTION_OPEN_CAR_CONTROL_PROJECTION` | `com.wmqc.miroot.car.ACTION_OPEN_CAR_CONTROL_PROJECTION` |
| `ACTION_STOP_CAR_CONTROL_PROJECTION` | `com.wmqc.miroot.car.ACTION_STOP_CAR_CONTROL_PROJECTION` |
| `EXTRA_CAR_PROJECTION_OP` | `com.wmqc.miroot.car.EXTRA_CAR_PROJECTION_OP` |
| Extra 内开始/停止 | `start` / `stop` |

### ADB 示例

```bash
# 一键切换（无 Extra：已在车控投屏则停，否则开）
adb shell am broadcast -a com.wmqc.miroot.car.ACTION_OPEN_CAR_CONTROL_PROJECTION com.wmqc.miroot
```


```bash
# 显式仅开始

  --es com.wmqc.miroot.car.EXTRA_CAR_PROJECTION_OP start \
  com.wmqc.miroot
```

```bash
# 停止：专用 Action（推荐）
adb shell am broadcast -a com.wmqc.miroot.car.ACTION_STOP_CAR_CONTROL_PROJECTION com.wmqc.miroot
```

```bash
# 显式接收器
adb shell am broadcast -n com.wmqc.miroot/com.wmqc.miroot.car.CarControlBroadcastReceiver \
```

## 五、激活状态查询（离线激活是否生效）

判断与应用内「离线激活」一致：已在本机校验并写入激活标记则为已激活。

| 项 | 值 |
|----|-----|
| 接收器（可显式 `-n`） | `com.wmqc.miroot.license.ActivationStatusQueryReceiver` |
| 清单 | `android:exported="true"` |

### 请求 Action（必填）

```
com.wmqc.miroot.license.ACTION_QUERY_ACTIVATION_STATUS
```

### 默认回执 Action

```
com.wmqc.miroot.license.ACTION_ACTIVATION_STATUS_RESULT
```

### 请求 Extra（可选）

| Key | 类型 | 说明 |
|-----|------|------|
| `requestId` | String | 回执中原样带回 |
| `replyAction` | String | 自定义回执 Action；不传则用默认 |

### 回执 Extra（MiRoot 发出）

| Key | 类型 | 说明 |
|-----|------|------|
| `requestId` | String | 与请求一致 |
| `success` | String | `"true"` / `"false"`（查询本身是否成功；当前实现成功时为 `"true"`） |
| `activated` | String | 是否已离线激活：`"true"` / `"false"` |

### 常量（与 `ActivationStatusIntents` 一致）

| 含义 | 字符串 |
|------|--------|
| 请求 Action | `com.wmqc.miroot.license.ACTION_QUERY_ACTIVATION_STATUS` |
| 默认回执 Action | `com.wmqc.miroot.license.ACTION_ACTIVATION_STATUS_RESULT` |
| 回执：已激活 | Extra 键 `activated`，值 `"true"` / `"false"` |

### ADB 示例

```bash
# 查询激活状态（回执默认发往 com.wmqc.miroot.license.ACTION_ACTIVATION_STATUS_RESULT）
adb shell am broadcast -a com.wmqc.miroot.license.ACTION_QUERY_ACTIVATION_STATUS \
  --es requestId q-act-1 \
  com.wmqc.miroot
```

```bash
# 显式接收器
adb shell am broadcast -n com.wmqc.miroot/com.wmqc.miroot.license.ActivationStatusQueryReceiver \
  -a com.wmqc.miroot.license.ACTION_QUERY_ACTIVATION_STATUS
```

### 应用内调用示例（Kotlin）

```kotlin
import android.content.Intent

fun queryMiRootActivationStatus(ctx: android.content.Context, requestId: String? = null) {
    ctx.sendBroadcast(
        Intent("com.wmqc.miroot.license.ACTION_QUERY_ACTIVATION_STATUS").apply {
            setPackage("com.wmqc.miroot")
            putExtra("requestId", requestId ?: "")
        },
    )
}
```

在己方应用中注册 `BroadcastReceiver`，`IntentFilter` 使用默认回执 Action（或发送请求时在 Extra `replyAction` 中指定与己方一致的 Action），读取 `activated`（`"true"` / `"false"`）。
